Mocha vs Jest.

Общее

Плюсы

  • одинаковый синтаксис для написания простых тестов

Минусы

  • и там и там оказалось весьма нетривиально настроить работу с import

Mocha

Плюсы

  • На больших тестах с множественными проверками отрабатывает быстрее
  • Есть различные предустановленные типы вывода результатов. Некоторые полезные, некоторые просто забавные: к примеру такой котик

В jest reporter тоже можно менять, но списка предустановленных я не нашла.

Минусы

  • Если настроить запуск тестов через скрипт в package.json то появляется интересный спецэффект - максимальное время на выполнение теста 2 с. Нормально отдебажить это явно помешает. Однако все работает нормально если запускать средствами ide. В Jest работают оба варианта
  • Надо подключать библиотеку для ассертов. К примеру chai

Jest

Плюсы

  • При создании через create-react-app предустанавливается. Не надо дополнительно настраивать
  • Умеет в многопоточность (исходя из документации)
  • есть expect из коробки
  • Тесты из разных файлов автоматически считаются разными test suites. Удобнее оценивать что конкретно сломалось.
  • Есть code coverage и снепшот тесты

Минусы

  •  Требует особого именования файлов с тестами. Зато это позволяет автоматически находить файлы с тестами. Так что плюс это или минус - неоднозначно.
  • Если по глупости( как у меня и случилось) не знать о наличие предустановленной версии, и установить вручную, можно получить конфликт версий и отказ ide запускать тесты с красивой визуализацией. Только через консоль. Удалось настроить при переустановке всех модулей, но это было неприятным моментом. (Конечно это больше вопрос к ide, но все же)
